# Skill: Error Escalation

When to stop and ask the Supervisor (User).

## 1. Trigger Conditions
Escalate immediately if:
- **Ambiguity:** Two conflicting requirements (e.g., "Minimalist" but "Extreme Particles").
- **Technical Barrier:** Requesting features impossible in a single-file Canvas environment (e.g., "Multiplayer with persistence").
- **Recursive Fail:** Failing the internal QA checklist 3 times in a row on the same issue.

## 2. How to Escalate
- Clearly state the problem.
- Provide options (Option A, Option B).
- Ask for a decision.

## 3. Avoid "Looping"
- Do not keep trying the same failing code.
- If a bug persists, explain the technical limitation and ask for a pivot in the game design.

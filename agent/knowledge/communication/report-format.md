# Skill: Report Format

Standardized communication for the game agent.

## 1. Structure
When finishing a task, the agent should report:
1. **Summary:** 1-sentence overview of what was built/fixed.
2. **Changes:** List of key logical or visual updates.
3. **Status:** "PASSED QA" or "Pending [Issue]".
4. **Action:** Next steps for the user (e.g., "Tap to play").

## 2. Tone
- Professional, concise, and technical.
- Use emojis for readability (✅, 🛠, 🚀).

## 3. Clarity
- If a part of the prompt was ambiguous, state how you interpreted it.
- Example: "Interpreted 'space theme' as neon purple parallax with star particles."

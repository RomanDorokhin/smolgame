# Архитектура SmolGame 🏛️

Этот документ описывает устройство платформы SmolGame, чтобы проект мог жить 10–100 лет и любой разработчик мог понять его за 30 минут.

## 1. Общая концепция
SmolGame — это децентрализованная (по духу) платформа для мини-игр в Telegram. 
- **Frontend**: Vanilla JS (без фреймворков), HTML5, CSS3. Максимальная легкость и скорость.
- **Backend**: Cloudflare Workers (Serverless) + D1 (SQL) + R2 (Storage). Нулевая стоимость поддержки при малых нагрузках.
- **Games**: Каждая игра — это изолированный HTML-файл, хостящийся на GitHub Pages.

## 2. Структура Frontend (папка `js/`)

Проект разбит на логические слои:

### 📂 `core/` (Ядро)
Низкоуровневые модули, от которых зависит всё остальное.
- `config.js`: Глобальные константы (API URL, Bot Username).
- `state.js`: Глобальный объект `STATE` и `USER`. Хранит текущую сессию.
- `api.js`: Прослойка для общения с Backend (Cloudflare Worker).
- `telegram.js`: Интеграция с Telegram WebApp API.

### 📂 `features/` (Бизнес-логика)
Крупные функциональные модули.
- `feed/`: Вертикальная лента (алгоритмы скролла, загрузка карточек).
- `profile/`: Личный кабинет, редактирование, просмотр чужих профилей.
- `game-launch/`: Логика запуска игр в iframe, передача параметров.

### 📂 `ui/` (Интерфейс)
Всё, что касается отрисовки и навигации.
- `screens.js`: Главный менеджер экранов. Переключает видимость слоев.
- `overlays/`: Всплывающие окна, поповеры, шторки (pull-to-close).

### 📂 `modules/` (Вспомогательные системы)
- `events.js`: Глобальная шина событий (клик по кнопкам, кастомные ивенты).
- `i18n.js`: Система локализации (Ru/En).

## 3. Жизненный цикл приложения (Bootstrapping)

1. **Загрузка**: `index.html` подгружает скрипты в строгом порядке (Core → Features → UI → Main).
2. **Инициализация**: `boot-splash.js` показывает заставку.
3. **Telegram Auth**: `telegram-initdata.js` проверяет подпись Telegram.
4. **State**: `state.js` подгружает данные о пользователе.
5. **Main**: `main.js` скрывает сплэш и открывает `Feed` (Ленту).

## 4. AI Архитектор (папка `agent/`)
Агент — это отдельное React-приложение (Vite), которое встраивается в платформу.
- Он живет в `iframe` или инжектится как модуль.
- Использует **Surgical Repairs** (Search/Replace блоки) для правки кода игр.
- Публикует игры через бэкенд-прокси на GitHub.

## 5. Принципы разработки
1. **No Dependencies**: Минимум сторонних библиотек. Только то, что можно скопировать одним файлом.
2. **Mobile First**: Всё управление рассчитано на тач-интерфейс Telegram.
3. **Fail Gracefully**: Если Telegram API недоступно, приложение должно работать как обычный веб-сайт.

# Архитектура SmolGame (VPS) 🏛️

Этот документ описывает текущую архитектуру развертывания проекта SmolGame на VPS (Ubuntu), после миграции с Cloudflare Workers/Pages.

## 1. Общая топология сервера

Проект работает на Ubuntu VPS сервере с публичным IP `89.167.94.140` и привязанным доменом `smolgame.ru`.

### Директории на сервере:

*   **/var/www/smolgame/frontend/** - Статика фронтенда (HTML, CSS, JS) и AI-агента. Обслуживается Nginx.
*   **/var/www/smolgame/games/** - Директория с локальными копиями игр (склонированными с GitHub). Обслуживается Nginx по пути `/g/`.
*   **/root/smolgame-frontend/backend/** - Исходный код бэкенда (Node.js) и база данных SQLite. Обслуживается PM2.

## 2. Веб-сервер (Nginx)

Nginx выступает в роли обратного прокси-сервера и файлового сервера:

*   **HTTPS/SSL**: Настроен сертификат от Let's Encrypt (`certbot`) для `smolgame.ru` и `www.smolgame.ru`.
*   **Фронтенд (`/`)**: Отдает статические файлы из `/var/www/smolgame/frontend/`.
*   **API (`/api/`)**: Проксирует запросы на локальный порт `3001` (Node.js бэкенд).
*   **Игры (`/g/`)**: Отдает статические файлы игр из `/var/www/smolgame/games/`.

## 3. Бэкенд (Node.js + PM2)

*   Бэкенд портирован с Cloudflare Workers на стандартный Node.js (исходники в `/root/smolgame-frontend/backend/src/`).
*   Запускается через файл `node-server.js` на порту `3001`.
*   Управляется менеджером процессов **PM2** (процесс `smolgame-api`).
*   **Переменные окружения**: Загружаются из файла `.env` в директории бэкенда (включает токены Telegram, GitHub и т.д.).

## 4. База данных (SQLite)

*   Данные мигрированы из Cloudflare D1 в локальную базу **SQLite**.
*   Файл базы: `/root/smolgame-frontend/backend/smolgame.db`.
*   Для работы используется адаптер `sqlite-db.js`, который эмулирует формат ответов D1 (возвращает `{ results: [...] }`), чтобы не переписывать логику маршрутов в `routes.js`.

## 5. Хранение игр

Вместо загрузки через iframe с GitHub Pages, игры теперь хранятся и раздаются локально:
*   Путь: `/var/www/smolgame/games/[GAME_ID]/`
*   В базе данных в таблице `games` поле `url` указывает на локальный HTTPS роут: `https://smolgame.ru/g/[GAME_ID]/`

## 6. Деплой и обновление (Памятка)

*   **Обновление фронтенда**: 
    1. Скопировать новые файлы в `/var/www/smolgame/frontend/`.
    2. В файле `js/core/api.js` переменная `PROD_API_BASE` должна быть установлена в `''` (пустая строка), чтобы API запросы шли на относительный путь `/api/...`.
*   **Обновление бэкенда**: 
    1. Сделать `git pull` в папке `/root/smolgame-frontend/backend/`.
    2. Перезапустить процесс: `pm2 restart smolgame-api`.
*   **Просмотр логов бэкенда**: `pm2 logs smolgame-api`

# SmolGame — дизайн-система и UI (документация)

Документ для команды и будущих правок: **что за чем**, **где лежит код**, **почему так сделано**. Цель визуала — не конкурировать с играми: спокойный тёмный хром, аккуратные акценты, понятные жесты в Telegram Mini App.

## 2. Принципы продукта

1. **Игра — главный герой.** В режиме discovery нижний хром и боковые действия — спокойные; в режиме **«Играть»** (`body.feed-game-focus`) платформенный UI скрыт, чтобы не отвлекать от iframe.
2. **Ориентиры — Telegram / минимализм YouTube:** много воздуха, мягкие границы, мелкие детали заметны только при внимательном взгляде (doodle-фон).
3. **Один язык контролов:** кнопки через систему `.sg-btn`, иконки — SVG с `currentColor`, без эмодзи в критичных местах.
4. **Жесты и iframe:** вертикальная лента — свайп по `#feed` (см. `FEED_VERTICAL` в `feed.js`). В discovery у активного слайда iframe с `pointer-events: none`, чтобы жест шёл на контейнер; в **режиме игры** pointer-events у iframe включены. Горизонтальный свайп в коде остаётся запасным для `FEED_VERTICAL = false`.
5. **Обратная связь:** короткие тосты, лёгкий haptic в Telegram, пустые и ошибочные состояния в одном визуальном семействе (`.sg-empty-state`).
6. **Где я в приложении:** компактная шапка `#app-tab-chrome` (марк + название раздела) на основных табах (`body.app-main-chrome`); в режиме фокуса на игре скрыта.

## 3. Цветовая система (`:root` в `styles.css`)

| Токен | Назначение |
|-------|------------|
| `--bg` | Фон приложения (`#0a0b0e`) |
| `--surface`, `--surface-elevated` | Карточки, поднятые панели |
| `--border` | Тонкие разделители |
| `--text`, `--muted` | Основной и вторичный текст |
| `--accent`, `--accent2`, `--accent-soft`, `--accent-rgb` | Холодный серо-голубой: ссылки, обводки фокуса, вторичные акценты. Не основной цвет заливки CTA — чтобы не «кричал». |
| `--cta-bg`, `--cta-border`, … | **Основные действия** (отправить, понятно): нейтральный графит, не конкурирует с игрой. |
| `--brand-1`, `--brand-2` | Градиент логотипа / вордмарка |
| `--like-on`, `--like-on-soft`, `--like-on-rgb` | Активный лайк: тёплый коралл, отдельно от `--accent` |

**Жанры:** классы `.sg-genre--arcade`, `.sg-genre--puzzle`, … и дочерние `.sg-gi-a` … для многоцветных SVG-групп в `genre-icons.js`.

**Загрузка игры:** две основные карточки `.method-card--url` и `.method-card--github` (сетка `.upload-methods--dual`); премиум — вторичная ссылка `.upload-more-btn` + `upload-show-premium` → форма `#form-premium`. Кнопки экрана загрузки — `.sg-btn` (в т.ч. GitHub OAuth, режим paste/files, сабмиты). Гайд из трёх шагов + FAQ — **ниже** форм; шаги `.help-acc--step1…3` с цветовыми акцентами.

## 5. Ключевые компоненты

### 5.1 Кнопки `.sg-btn`

Модификаторы: `--primary`, `--secondary`, `--ghost`, `--icon`, `--fab`, **`--screen-cta`** (primary чуть контрастнее на экранах без превью игры — пустые состояния, формы, оверлеи).  
Единые радиусы `--btn-radius-md/lg`, отступы `--btn-pad-x/y`.  
Активное состояние: лёгкий `scale` + смена фона (см. `.sg-btn:active`).

### Панели в стиле Telegram (стекло)

Токены `--tg-glass-*` в `:root` и классы **`.tg-glass`**, **`.tg-glass--sheet`** (нижний лист чуть плотнее), **`.tg-glass--muted`** (без внешней тени). Используются для: `#feed-reviews-drawer`, карточек `#feed-nav-tip-overlay`, welcome ленты (`.feed-onboarding-welcome-inner`), welcome загрузки, `tg-only-card`, `onboarding-card` — один blur/граница, чтобы модалки не «бились» друг с другом.

### 5.2 Бренд `.sg-brand`, `.sg-brand-mark`

Марка — геометрия в SVG (без текста внутри SVG), чтобы не зависеть от шрифта внутри вектора.  
Варианты размера: `--sm`, `--xs`; вращение только там, где нужна загрузка: `.sg-brand-mark--spin` / `--loader` (splash по умолчанию **без** спина).

### 5.3 Лента и слои

- `#feed` — вертикальные полноэкранные слайды; внутри слайда: `.slide-inner` → `.slide-stack` (плейсхолдер + `.slide-game` iframe) и при вертикальной ленте оверлей **`.slide-discovery`** (название, короткое описание, жанр, CTA **«Играть»**).
- `#side-actions` — лайк, шаринг, жалоба; скрыты в `feed-game-focus`.
- `#game-info` — полоса автора внизу (в фокусе игры скрыта); дублировать название игры в блоке автора не нужно.
- `#dots` — индикатор позиции в ленте; в фокусе игры скрыты.
- `#feed-exit-focus` — кнопка «← Лента» поверх игры; показывается только в `feed-game-focus`.
- **Устарело в UI:** `#swipe-strip`, стрелки навигации по ленте, `feed-coach-fab` — удалены; не возвращать без явной продуктовой задачи.

### 5.4 Навигация и библиотека

- Нижний бар `#bottom-nav`: **Лента** · **Игры** (сердечко) · **Загрузить** (центральная кнопка) · **Поиск** · **Профиль**.
- `#games-library-screen` — оверлей «Игры»: секции **«Лайкнул»** и **«Играл»**; закрытие свайпом вниз, повторным табом или кнопкой **«← Лента»** в шапке.
- Оверлеи профиля, поиска, автора, загрузки — кнопка **«← Лента»** (или **«← Назад»** у автора) в шапке; дополнительно — таб или pull-to-close где подключено.

### 5.5 Подсказка ленты (первый заход)

- Оверлей `#feed-nav-tip-overlay`: сначала краткий текст + демо (SVG-указатель, без эмодзи), ссылка **«Подробная справка»** открывает вторую карточку с буллетами; логика в `feed-nav-tip.js` (`STORAGE_KEYS.feedNavTip`, класс `feed-nav-tip--detail` на оверлее).
- Первый запуск ленты (`feed-onboarding.js`): один экран welcome + **«Начать»**, без анимации демо-свайпа; после закрытия при необходимости показывается краткая подсказка ленты.
- После свайпа по ленте: `markFeedSwipeLearned` + снятие tease-коучинга в `feed.js` (локально `feedSwipeLearned`).
- Tease «дёрни ленту»: класс `feed-swipe-tease-burst` на `body` из `feed.js` (для вертикали часть старых веток отключена — см. `canRunFeedSwipeTeaseBurst`).

### 5.6 Фон «doodle»

Переменная `--sg-doodle` (inline SVG data URL) + `::before` на `body` / экранах / `#feed` (см. `styles.css`). Низкая непрозрачность — фон «простой», детали при присмотре.

### 5.7 Пустые и ошибочные состояния

- **Лента пустая:** `#empty-state` → блок `#empty-state-default` (`.sg-empty-state`).
- **Ошибка загрузки ленты:** `#empty-state-error`, текст из `userFacingError`, кнопка «Повторить» → `data-action="feed-retry"`.
- **Сетки профиля / автора:** `sgEmptyGridHtml()` в `utils.js` + `.sg-empty-state--grid`.

### 5.8 Тосты и тактильный отклик

- `showToast(msg)` — короткие фразы; длинные ошибки режутся в `userFacingError`.
- `hapticLight` / `hapticSuccess` / `hapticWarning` — обёртки над `Telegram.WebApp.HapticFeedback`.
- Переключение таба: лёгкий haptic в `switchTab`.

## 8. Telegram Mini App

- `viewport-fit=cover`, safe-area: `--safe-top`, `--safe-bottom` в расчётах высоты игры и навбара.
- `telegram-web-app.js` в `index.html`; гейт вне Telegram — `tg-only-wall`.
- На **первом** слайде ленты при вертикальном жесте вниз по `#feed` (достаточно сильный downward drag) обработчик в `feed.js` **отпускает** захват указателя, чтобы жест мог уйти в нативное закрытие WebView (см. `TELEGRAM_PASS_DOWN_PX`). В режиме **«Играть»** свайп ленты отключён — риск случайного закрытия ниже.
- Вордмарк SmolGame для скачивания из профиля **убран**; бренд в сплэше и шапке табов остаётся по продуктовой логике.

## 9. Как безопасно менять дизайн

1. **Цвета/радиусы** — по возможности только `:root` и существующие классы.
2. **Новый экран** — те же `.sg-btn`, `.sg-empty-state`, те же отступы что у соседних экранов.
3. **Новые иконки** — SVG, `stroke="currentColor"`, без эмодзи в кнопках навигации.
4. **После выкладки** — поднять `?v=` в `index.html` для сброса кэша WebView.
5. **Свайп / iframe** — в discovery не включать `pointer-events` на iframe; в режиме игры — наоборот. Не вешать агрессивный `touch-action` на всё дерево `#feed` без необходимости (см. комментарии в `styles.css` и `feed.js`).

## 10. История решений (кратко)

- Ушли от агрессивного красно-чёрного к **холодному акценту + нейтральным CTA** — меньше отвлечения от контента.
- **Лайк** выделен тёплым цветом — отдельная семантика «эмоция», не путать с системным акцентом.
- **Вертикальная лента + «Играть»** — discovery отделён от полноэкранной игры; свайп по полоске убран как шумный и путавший с жестами Telegram.
- **Doodle-фон** — лёгкая индивидуальность без шума.
- **Документация** — чтобы новый человек в проекте не ломал эти ограничения нечаянно.

При расхождении кода и этого файла **источник правды — репозиторий**; при крупных UI-изменениях имеет смысл обновить разделы 5–7.

# Руководство разработчика игры под SmolGame

Игра запускается **внутри Telegram Mini App** во встроенном браузере (WebView): страница игры открывается в **`<iframe>`** поверх ленты или в полноэкранном режиме после нажатия **«Играть»**. Ниже — правила, чтобы игра стабильно работала у игроков и проходила модерацию.

### Использование этого файла с ИИ

Если вы поручаете ИИ написать или доработать игру под SmolGame, **вставьте в запрос целиком этот документ** (или ссылку на актуальный файл в репозитории) и добавьте явную инструкцию, например:

> Ты помогаешь с HTML5-игрой для SmolGame. Соблюдай **все** требования из приложенного руководства разработчика: запуск в iframe внутри Telegram WebView, HTTPS, пути для GitHub Pages, тач-ввод, раздел «Техническое задание на модерацию» и чеклист. Не предлагай обходить ограничения платформы. Если что-то противоречит руководству — следуй руководству.

ИИ должен трактовать раздел **«Техническое задание на модерацию»** как обязательный контракт: отклонения возможны и при нарушении пунктов модерации, даже если форма отправки на сервер прошла.

---

## 1. Как устроен запуск

| Режим | Описание |
|--------|----------|
| **Лента** | На карточке видно превью; игровой `iframe` может быть без фокуса — управление лентой свайпом. |
| **Фокус («Играть»)** | Полный экран под игру, свайп ленты отключён; выход — **«← Лента»**. |

Песочница iframe (важно для API браузера):

- Разрешены скрипты, формы, всплывающие окна, **`allow-same-origin`** — чтобы работали **`localStorage` / `sessionStorage`** и обычный origin страницы.
- Игра по-прежнему загружается с **вашего домена** (GitHub Pages и т.д.), не с домена SmolGame.

**Рекомендация:** любые обращения к `localStorage`, `indexedDB`, сети оборачивайте в `try/catch`, чтобы при ограничениях в старых WebView игра не падала целиком.

---

## 2. URL и GitHub Pages

Игра часто живёт по адресу вида:

`https://ВАШ_НИК.github.io/ИМЯ_РЕПО/`

Это **не корень** домена `github.io`, а **подпапка проекта**.

### Обязательно

- Подключайте скрипты, стили, картинки **относительными путями**: `./game.js`, `./assets/bg.png`.
- Или задайте в `<head>`: `<base href="./">` или базовый URL каталога игры.

### Нельзя

- В проектном Pages использовать пути от корня сайта без учёта репозитория:  
  `src="/game.js"` → браузер запросит `https://ВАШ_НИК.github.io/game.js` вместо файла из репозитория → **404 и чёрный экран**.

Перед отправкой на модерацию откройте игру **по тому же URL**, который укажете в карточке (из браузера, лучше с телефона).

---

## 3. Вёрстка и экран

- **`viewport`:** `width=device-width`, `initial-scale=1`, при необходимости `viewport-fit=cover` под вырезы.
- **Высота:** используйте `100vh` / `100dvh` / `100svh` осознанно; в WebView высота может меняться при появлении клавиатуры.
- **Фон:** если корень страницы чёрный — при ошибке скрипта пользователь увидит «просто чёрный экран»; для отладки временно можно другой фон или текст «loading».

---

## 4. Ввод

- Рассчитывайте на **тач** и отсутствие мыши.
- **`touch-action`** и **`preventDefault`** на жестах — только там, где нужно; не блокируйте скролл без необходимости.
- Не полагайтесь только на клавиатуру (на телефоне её может не быть).

---

## 5. Звук

Автовоспроизведение звука в браузерах часто требует **жеста пользователя**. Запускайте аудио после первого тапа или явной кнопки «Старт / Включить звук».

---

## 6. Производительность

- Цель — плавная работа на **средних телефонах**.
- Ограничивайте разрешение canvas, тяжёлые постобработки и число одновременных объектов.
- Не подгружайте десятки мегабайт ресурсов при первом кадре.

---

## 7. Сеть и безопасность

- Только **HTTPS** для загрузки игры и ресурсов.
- Не смешивайте **HTTP**-ресурсы на HTTPS-странице (mixed content может блокироваться).
- Внешние скрипты и произвольный пользовательский HTML внутри игры — ваша ответственность; модерация может отклонить карточку при подозрительном поведении.

---

## 8. Чего избегать

- Уход со страницы игры через **`top.location`** без нужды — ломает опыт внутри мини-аппа.
- Бесконечные редиректы и открытие сторонних доменов вместо игры.
- Требование только десктопного браузера или плагинов.

---

## 9. Чеклист перед публикацией

1. Игра открывается по **точному HTTPS URL**, который внесёте в карточку.
2. На телефоне в обычном браузере всё видно и управляется пальцем.
3. Нет ошибок в консоли (особенно из‑за путей к `*.js` / `*.css`).
4. `localStorage` / сохранения не роняют игру при ошибке (`try/catch`).
5. Для GitHub Pages проверены **относительные** пути ко всем ассетам.
6. Выполнены пункты **«Техническое задание на модерацию»** ниже (не только технические разделы 1–8).

---

## 10. Техническое задание на модерацию

Модерация в SmolGame — **два уровня**: (1) проверки при отправке карточки на сервер и (2) **ручное** решение модератора по очереди. Прохождение формы не гарантирует публикацию.

### 10.1. Карточка игры (обязательные поля и лимиты)

| Поле | Условие |
|------|--------|
| **Название** | Обязательно, до **40** символов (лишние обрезаются при сохранении). Должно соответствовать игре, без вводящего в заблуждение кликбейта. |
| **Описание** | До **120** символов. Кратко и честно: что игрок делает в игре, без обещаний функций, которых нет. |
| **Жанр** | Один из списка платформы: Аркада, Головоломка, Экшен, Казуалка, Стратегия, Гонки, Платформер, Прочее. Иное значение сохранится как **Прочее**. |
| **Эмодзи жанра** | До **8** символов (обычно один эмодзи). |
| **URL игры** | Только **`https://`**. Должен открывать **непосредственно играбельную страницу** (не пустой репозиторий, не README, не редирект на магазин/донат вместо игры). Для **GitHub Pages** хост должен быть **`*.github.io`** (например `https://username.github.io/` для user pages или `https://username.github.io/repo-name/` для project pages). Любой другой **валидный HTTPS**-URL тоже подходит для отправки формы, если по нему грузится игра. |
| **Обложка** | Если указана — только **`https://`** ссылка на изображение (корректный URL). Загрузка через приложение: только изображение, **до 2 МБ**. Обложка должна относиться к игре, без шок-контента и обмана. |

Несоответствие таблицы → заявка не примется или карточка будет отклонена при проверке.

### 10.2. Технические условия для прохождения модерации

Считается **нарушением ТЗ**, если при проверке по ссылке из карточки:

- Игра **не запускается** или показывает пустой/битый экран из‑за ошибок загрузки (в т.ч. неверные пути к скриптам на GitHub Pages — см. разделы **2**, **9**).
- Не соблюдены **HTTPS** и отсутствие **mixed content** (раздел **7**).
- Управление **не работает на таче** / игра требует только клавиатуру или плагины (разделы **4**, **8**).
- Игра **ломает контейнер** мини-аппа: принудительный уход с страницы через `top.location`, бесконечные редиректы, подмена URL на сторонний сервис вместо игры (раздел **8**).
- **Вредоносное или мошенническое поведение**: фишинг, скрытый майнинг, воровство данных, обманная монетизация, контент для взрослых в детском контексте, экстремизм, очевидный спам — **отклонение** без обязательного перечисления каждого случая в правилах.
- **Внешние скрипты и контент** ведут себя подозрительно или противоречат заявленному жанру/описанию (раздел **7**).

Рекомендации из разделов **1–8** (viewport, звук, производительность и т.д.) снижают риск отклонения из‑за плохого опыта на реальных устройствах.

### 10.3. Условия, которые проверяет модератор вручную

Модератор опирается на **это ТЗ** и здравый смысл безопасности каталога. Обычно проверяют:

1. **Соответствие карточки игре** — название, описание, жанр и обложка не вводят в заблуждение.
2. **Работоспособность** — игра проходима или осмысленно играбельна в WebView Telegram на типичном телефоне.
3. **Безопасность и уместность** — контент и поведение приемлемы для публичной ленты мини-игр.
4. **Отсутствие злоупотреблений** — не дубликат чужой игры без добавленной ценности, не чистая реклама стороннего продукта под видом игры.

**Спорные случаи** решает команда площадки; решение модератора по очереди финальное в рамках текущих правил сервиса.

### 10.4. После публикации

Изменение **названия, описания, жанра или обложки** в профиле снова отправляет карточку **на модерацию**; **URL игры** через эту правку не меняется — если нужна новая ссылка, уточняйте у поддержки/правил продукта.

# SmolGame — спецификация для разработчика

Документ для **автора HTML5-игры**, которую вешают в ленту SmolGame (Telegram Mini App). Ниже: как устроен запуск, как собрать билд под WebView и **какие условия должны быть выполнены**, чтобы карточка прошла модерацию и не отваливалась у игроков.

Игра грузится **с вашего HTTPS-origin** (GitHub Pages и т.п.) во **`iframe`** внутри WebView: в ленте превью, по «Играть» — полноэкранный режим, выход — «← Лента».

---

## 1. Запуск в приложении

| Режим | Поведение |
|--------|-----------|
| **Лента** | Превью на карточке; `iframe` может быть без фокуса — свайп ленты. |
| **«Играть»** | Полный экран под игру, свайп ленты отключён. |

Песочница `iframe`: скрипты, формы, попапы, **`allow-same-origin`** — работают **`localStorage` / `sessionStorage`** и ваш обычный origin.

**Практика:** обращения к `localStorage`, `indexedDB`, сети — в `try/catch`, чтобы старый WebView не ронял игру целиком.

---

## 2. URL и GitHub Pages

Типичный адрес project pages:

`https://ВАШ_НИК.github.io/ИМЯ_РЕПО/`

### Обязательно

- Ассеты — **относительные пути**: `./game.js`, `./assets/bg.png`, или `<base href="./">` в `<head>`.

### Нельзя

- `src="/game.js"` в project pages → запрос уйдёт на `https://ВАШ_НИК.github.io/game.js` → **404**, чёрный экран.

Перед отправкой карточки откройте игру **по тому же URL**, что пойдёт в форму (с телефона — обязательно).

---

## 3. Вёрстка и экран

- `viewport`: `width=device-width`, `initial-scale=1`, при необходимости `viewport-fit=cover`.
- Высота: `100vh` / `100dvh` / `100svh` — учитывайте клавиатуру в WebView.
- Чёрный фон без индикации загрузки маскирует ошибки скрипта — для отладки полезен контрастный фон или «loading».

---

## 4. Ввод

- **Тач** по умолчанию, мыши может не быть.
- `touch-action` / `preventDefault` — точечно, не ломайте скролл без нужды.
- Не делайте игру только под клавиатуру.

---

## 5. Звук

Автоплей часто блокируется — стартуйте звук после первого тапа или кнопки «Старт / Звук».

---

## 6. Производительность

- Ориентир — **средний телефон**: лимит canvas, постобработка, число объектов.
- Не тащите десятки МБ на первый кадр.

---

## 7. Сеть и безопасность

- Игра и ресурсы — **HTTPS**; без mixed content (HTTP на HTTPS-странице).
- Внешние скрипты и UGC — на вашей ответственности; при подозрительном поведении карточку не пропустят.

---

## 8. Антипаттерны под iframe / Mini App

- Ломать оболочку: **`top.location`**, бесконечные редиректы, подмена страницы на не-игру.
- Только десктоп / плагины.

---

## 9. Чеклист перед отправкой карточки

1. Тот же **HTTPS URL**, что в форме — открывается и играется.
2. Телефон, палец — всё читается и управляется.
3. Консоль без ошибок загрузки `*.js` / `*.css`.
4. `localStorage` и т.п. не валят игру без `try/catch`.
5. На GitHub Pages — относительные пути ко всем ассетам.
6. Выполнены требования раздела **«Модерация»** ниже.

---

## 10. Модерация

Два этапа: **валидация на сервере** при submit и **ручная** очередь. Успешный submit ≠ публикация.

### 10.1. Карточка (поля и лимиты API)

| Поле | Условие |
|------|--------|
| **Название** | Обязательно, до **40** символов (обрезка на сервере). Соответствует игре, без обмана в заголовке. |
| **Описание** | До **120** символов. Фактическое действие в игре, без выдуманных фич. |
| **Жанр** | `Аркада`, `Головоломка`, `Экшен`, `Казуалка`, `Стратегия`, `Гонки`, `Платформер`, `Прочее` — иначе сохранится **`Прочее`**. |
| **Эмодзи жанра** | До **8** символов. |
| **URL игры** | Только **`https://`**. Открывается **играбельная** страница (не пустой репо, не README вместо игры, не редирект «купи / донать» вместо геймплея). Для GitHub Pages — хост **`*.github.io`** (`https://user.github.io/` или `https://user.github.io/repo/`). Любой другой валидный HTTPS допустим, если по нему реально грузится игра. |
| **Обложка** | Если есть — `https://` на картинку. Загрузка в приложении: только image, **≤ 2 МБ**. Соответствует игре, без токсичного/вводящего в заблуждение контента. |

Несоответствие — отказ уже на submit или при ручной проверке.

### 10.2. Технические причины отклонения билда

По ссылке из карточки:

- Не грузится / битый экран (в т.ч. неверные пути — см. §2, §9).
- Нет HTTPS или есть mixed content (§7).
- Нет нормального **тача**, только клавиатура/плагины (§4, §8).
- Ломается контейнер мини-аппа: `top.location`, редирект-ад, подмена на не-игру (§8).
- Вредоносность, фишинг, майнинг, вор данных, мошенническая монетизация, недопустимый контент для ленты, спам — **отклонение**.
- Внешние скрипты/контент ведут себя подозрительно или не бьются с описанием (§7).

Рекомендации §1–8 снижают риск отклонения из‑за плохого UX на реальных девайсах.

### 10.3. Ручная проверка

- Карточка **совпадает** с тем, что по ссылке.
- Игра **осмысленно играбельна** в Telegram WebView на типичном телефоне.
- **Безопасность и уместность** для публичной ленты.
- Не голый репост чужой игры без ценности, не реклама под видом игры.

Спор — команда площадки; решение по очереди финально в рамках правил сервиса.

### 10.4. Правка после публикации

Смена **названия, описания, жанра, обложки** в профиле → снова **на модерацию**. **URL игры** этой формой не меняется; другая ссылка — по правилам продукта / поддержке.

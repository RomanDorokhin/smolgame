SmolGame Премиум

Для команд с коммерческой тайной: исходный код игры хранится на защищённой стороне SmolGame, а не в открытом репозитории GitHub. Плюс ранние функции платформы и приоритет. Раздел в активной разработке.

Чем отличается от GitHub и «Ссылка»

Во вкладках Ссылка и GitHub игра живёт по публичному URL: код обычно лежит в открытом репозитории или на стороннем хостинге — любой может его скачать и изучить. В Премиуме планируется сценарий, когда билд или исходники лежат на закрытой инфраструктуре SmolGame: доступ только у сервиса и у тебя как автора, без публичного репозитория.

Кому это нужно

  • Коммерческие проекты и ноу-хау, которые нельзя выкладывать в открытый Git.
  • Кто хочет хостинг игры без обязательной публикации кода в github.io.
  • Ранний доступ к экспериментальным возможностям мини-аппа и поддержка продукта.

Статус

Приватное хранение и полный премиум-пайплайн ещё подключаются. Сейчас игры в ленту можно добавить через Ссылка или GitHub — так работает у всех; премиум откроет отдельный путь, когда будет готов.

У тебя включён ранний доступ (аккаунт в списке на сервере). Здесь появятся шаги загрузки в закрытое хранилище, как только подключим сценарий на бэкенде.

Ранний список. Чтобы попасть в allowlist премиум-тестеров, напиши владельцу бота / проекта — в Worker добавляется твой Telegram numeric id в PREMIUM_TG_IDS. Пока список закрыт, всё равно можно пользоваться Ссылка и GitHub без ограничений.

Как добавить игру: четыре шага (открывай по одному). Форма — под карточками «Ссылка», «GitHub» и при доступе — «Премиум» в одном ряду.

1 Создай игру с ИИ

Зайди на claude.ai или chatgpt.com — оба бесплатны. Напиши (можно скопировать):

Напиши мне простую браузерную игру на HTML + JavaScript.
Один файл, без внешних библиотек.
Жанр: [змейка / тетрис / кликер / своё]
Игра должна работать на мобильном телефоне.

ИИ выдаст код — скопируй целиком. Не понравилось — попроси переделать.

2 Выложи на GitHub Pages

Бесплатный хостинг от GitHub, без карты и серверов.

  1. github.com → регистрация (если нет)
  2. + → New repository, имя например my-game, Public → Create
  3. Add file → Create new file → имя index.html → вставь код → Commit changes
  4. Settings → Pages → Branch: main → Save

Через 1–2 мин ссылка вида https://твой-ник.github.io/my-game — это URL игры.

3 Скопируй ссылку

В Settings → Pages зелёная плашка: Your site is live at https://… — скопируй её. Подожди 1–3 мин после публикации, если не открывается сразу.

4 Заполни форму над этим блоком

Ссылка — URL игры уже на хостинге (например GitHub Pages). GitHub — привязка аккаунта, код или файлы уходят в твой репозиторий и Pages; каждый Telegram-аккаунт подтверждает доступ в GitHub отдельно. Премиум — для аккаунтов из списка на сервере: цель — хранить код на закрытой стороне SmolGame, а не в открытом репозитории (когда сценарий будет включён в приложении).

FAQ

Это правда бесплатно?

Да. У Claude и ChatGPT есть бесплатные режимы. GitHub Pages бесплатен.

Нужно уметь программировать?

Нет. Ты описываешь желаемое ИИ — он пишет код.

Игра не работает — что делать?

Скопируй текст ошибки из браузера и отправь ИИ: «Вот ошибка, исправь код».

Какие игры подходят?

Браузерные на HTML, один файл index.html, без серверной части.

Можно обновить игру после публикации?

Да. Измени index.html в репозитории — через пару минут обновится по той же ссылке.

На телефоне перетаскивание в игре «залипает»

Частая причина: touchstart на маленьком canvas, а touchmove/touchend только на document. В iOS и в iframe жест рвётся. Делай через Pointer Events и setPointerCapture на элементе, с которого начали драг, или вешай move/end на window с capture: true и обрабатывай touchcancel.

Почему GitHub?

Простой и надёжный бесплатный вариант: без карты и сложной настройки.
